# Live DB Security Audit — 2026-07-05

## Verdict: **NO** — the live RLS posture is not safe for adversarial traffic this week

The blocking issue is **DI-01**: `user_foods` can be INSERTed already `verification_status: 'verified'` via the anon key + raw PostgREST, bypassing the app route entirely and the BEFORE-UPDATE-only status-transition guard. That single hole lets any hostile authenticated user poison `verified_food_canonical` — the table every user's barcode lookup consults first — with plausible-but-wrong macros carrying a "verified" trust badge. Given project memory marks the nutrition trust-label cluster as CRITICAL and never-defer, this is a live, exploitable, one-request attack with no admin gate in between, and it must be fixed (a BEFORE INSERT trigger mirroring the existing `profiles` INSERT lockdown) before the traffic ramp widens.

Two more release-blocking items compound the picture, though they are functional rather than exploit-security: **SEC-04 / DI-03** — the `recipe_collections` migration (ENG-1126, merged to main, shipped in both web and mobile clients) was never pushed to the live database, so the shipped Library/Collections feature throws `relation does not exist` for every production user right now. That is a `supabase db push --linked` away from fixed, but it is currently broken in prod.

Beyond the two release-blockers, there is a real but non-blocking cluster of RLS/grant gaps that let hostile authenticated users write to or read shared/community tables outside their intended validated path — `barcode_mappings` direct-write (SEC-03/DI-02), draft recipe content leaking through `recipe_ingredients`/`recipe_steps` (SEC-02), and per-user vote visibility (SEC-10) — plus a real but slower-burning cost-abuse vector from confirmations-off, client-direct signup stacked against per-userId AI quotas (SEC-01/DI-04). None of these are as urgent as DI-01, but none should sit past this launch week either.

**Fix order:** DI-01 (today) → SEC-04/DI-03 migration push (today) → SEC-03/DI-02 barcode_mappings lockdown → SEC-02 recipe child-table RLS → SEC-01/DI-04 signup hardening (GoTrue config, owned by Grace) → the remaining severity-2/1 hardening items on a normal cadence.
