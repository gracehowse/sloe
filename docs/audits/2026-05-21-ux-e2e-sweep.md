## UX rubric notes (visual)

**Readability:** Cook step body uses `text-foreground` on `bg-background` (regression guarded). Progress “130% over target” red ring is legible on mobile.

**Layout:** Mobile Discover/Progress feel full-width; web Progress/Today use widened `product-shell`. Plan add-slot chips are compact on mobile (single row).

**Hierarchy:** Today FAB and serif “Today” title dominate; macro bar fills are very subtle (low contrast) — acceptable but watch in dark mode.

**States:** Empty Today “Start your day” + goal kcal clear on web and mobile.

**Parity:** Web desktop sidebar vs mobile bottom tabs is **intentional**; mobile tab label “Recipes” vs web “Discover” naming drift — document for sync-enforcer.

**Motion:** Not fully exercised; no blocking issues from static captures.
