What changed

  • #23.1 REPLACE Replaced the flat-dark-grey "0 Recipes" + "0 Streak" tiles with the outlined-coloured-tile pattern from Profile dark (DC14 Allowed borrow). Each tile carries its own accent: Recipes = amber outline + amber-soft background + amber digit, Streak = pale-indigo outline + pale-indigo-soft background + pale-indigo digit. Preserves the outlined-tile visual language across both Settings and Profile dark surfaces. What it duplicates / weakens: nothing — removes the DC14 adjacency erosion (where two adjacent dark surfaces in the same nav flow rendered stat tiles in different visual languages). The DC14 reference doc explicitly encourages replicating the outlined-coloured-tile + amber-warning pattern to other dark cards.

DC14 preservation check

DC14 (profile dark mode — outlined coloured tiles + amber safety-floor warning) is the strongest dark surface in the app. The bet is that outlined (not filled) coloured tiles on near-black canvas read as both premium and informative. Settings was breaking this bet by using a different tile pattern in an adjacent surface. The fix replicates Profile dark's tile shape so the visual language is unified. defended-choices.md §DC14 Allowed borrows explicitly lists this replication as encouraged.

Accent colour choice rationale

Profile dark uses green / indigo / amber / pink for kcal / protein / carbs / fat — each tied to a specific macro. For Settings, the tiles are non-macro stats, so the colours should NOT collide with the macro palette. Recipes = amber (associated with cooking / warmth, not collision with macro carbs because the carb tile uses orange not amber). Streak = pale-indigo (calm, matches DC8 streak-pip blue without shouting). Open question for Grace: do the colours match her brand intuition or should they shift?

Refused borrows considered

Considered making the tiles tappable to drill into the Recipes library or streak history. Refused for this sweep — that's a different proposal (navigation affordance) and would touch the routing layer. Pure visual replication is the DC14 Allowed borrow.
