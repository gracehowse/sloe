What changed

  • #3.1 TIGHTEN Hero gradient panel vertical padding compressed ~30% (96px top / 80px bottom → 56px / 48px). Brings the Monthly/Annual toggle into the first 900px viewport without scroll, exposing the actual decision point above the fold. What it duplicates / weakens: nothing — removes empty space, no content.
  • #3.2 NEW Inline "Includes VAT" line directly under the £59.99 / £7.99 price digit (DC15 allowed borrow from Stripe Checkout — "inline VAT line directly under the price digit"). UK/EU surfaces only — region-gated via existing regionCurrency detection. What it duplicates / weakens: nothing — adds 12px of visible-viewport text that previously wasn't there at all. **Resolves RTP-9 (active DC15 risk).**
  • #3.3 SUBTRACT Removed the small "SUPPR" badge floating at the bottom of the hero gradient panel. The top-bar already carries "Suppr" 60px above; this floating mini-badge was duplicating the wordmark for no functional reason. What it duplicates / weakens: duplicates the top-bar Suppr wordmark — competing instances of the brand mark in the same viewport.
  • #4.1 TIGHTEN Same hero-padding compression applied at mobile viewport (≤768px) so the £7.99 Pro price reads in the first mobile viewport without scroll. Inherits from #3.1; no separate styling. What it duplicates / weakens: nothing — subtractive.
  • #4.2 NEW Mobile-viewport inherit of #3.2 — same inline "Includes VAT" line, same region gate. What it duplicates / weakens: nothing — inherited.

DC15 preservation check

DC15 (UK/EU VAT-inclusive pricing posture) — load-bearing differentiator is the visible "VAT-inclusive" disclosure that most US-headquartered competitors get wrong. Pre-fix, the bet was real in code (Stripe Tax inclusive mode) but invisible to the user — DC15 explicitly listed this as an active risk. Post-fix, the "Includes VAT" line is rendered directly under the price digit in the first viewport. This is the exact Stripe Checkout allowed-borrow listed in defended-choices.md §DC15. The compliance + trust signal now lands.

DC4 / DC12 preservation

Trust chips (DC4) unchanged — three explicit guarantees retained verbatim in their current placement above the tier cards. No 4th "No price hikes ever" chip added. DC12 calm voice intact — hero copy is descriptive ("The full meal planning loop"), not shouty growth copy.

Refused borrows considered

Considered widening the gradient hero panel to full-bleed (Linear's pricing pattern). Refused — the bordered hero reads as "a card on a page" and matches the rest of Suppr's design language; full-bleed would over-promise marketing energy at a surface that's already meant to convert (not impress). Stripe / Linear / Calm all use bordered hero panels on pricing.

Capture gaps still open

Dark-mode pricing pixels exist (web-desktop-dark-pricing.png, web-mobile-dark-pricing.png) and validated the same gradient. After-state in dark would mirror the after-state shown here with dark canvas; no dark-specific concerns surfaced.
