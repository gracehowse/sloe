What changed

  • #5.1 REPLACE Replaced the blue rounded-square clipboard-with-checkmark placeholder icon with the Suppr "S" wordmark tile (same mark as mobile Welcome). Brand consistency at the first auth touchpoint. What it duplicates / weakens: nothing — replaces a stock placeholder with the real brand mark. Resolves RTP-10.
  • #5.2 SUBTRACT Removed the page-level "Sign in to Suppr" H1 + "Welcome back. Your Today, plan, and recipes are waiting." subtitle. The card directly below already carries "Welcome back / Sign in to continue." What it duplicates / weakens: duplicates the card title/subtitle — two competing headlines for the same surface.
  • #5.3 SUBTRACT Removed the in-card "Sign up / Sign in" segmented toggle. Sign-in is the only path through this surface; signup routes to /signup via a small text link at the bottom. What it duplicates / weakens: duplicates /signup route. The toggle made /login double as a signup entry, blurring the URL contract.
  • #5.4 SUBTRACT and #6.1 SUBTRACT Collapsed /login and /signin to one canonical URL with the other 301-redirecting. Picking /login as canonical (more common in SaaS sign-in patterns and likely better-indexed). Resolves RTP-5. What it duplicates / weakens: two URLs rendering the same form — full duplicate route.
  • #7.1 REPLACE Replaced /signup content with a real signup form: First name + Email + Password + "Continue with Apple" SSO + Create account primary CTA + "Already have an account? Sign in" secondary link. Mirrors the login card structure for visual consistency. What it duplicates / weakens: replaces the duplicated landing-page hero that was occupying /signup. Resolves RTP-1 (EMBARRASSING tier).
  • #7.2 SUBTRACT After 7.1 lands: /signup stays as its own surface (not a redirect to /login) — password managers + email-confirm flows need a dedicated signup URL to attach to. What it duplicates / weakens: nothing — preserves the route shape required by browser password managers.
  • #9.2 REPLACE Unauthed users hitting /home directly (e.g., from a shared link) now redirect to / (public landing) instead of /login (forced login wall). What it duplicates / weakens: removes a forced-login redirect that was treating link-share arrivals as suspicious. Resolves RTP-4.

DC4 / DC12 preservation

No DC4 (trust chips) change — trust chips live on paywall, not auth surfaces. DC12 (calm voice) preserved throughout — "Eat well, without overthinking it." reused on the signup card sub, matching the mobile Welcome headline (cross-platform voice unity).

Refused borrows considered

Considered adding a "social proof" line above the auth cards ("Join thousands tracking smarter" — the same line that gets removed from mobile Welcome in row 10.1). Refused — DC12 calm voice rejects unsubstantiated growth puffery. Same rationale as why 10.1 is a subtractive row.
