ENG-795 One soft elevation

BEFORE: results are a flat list separated by 1px hairlines — the same dividers used everywhere, no surface hierarchy. AFTER: grouped result cards carry one soft elevation; rows are seamed with a faint inset, not a hairline-as-divider. The tap target now reads as a card you press.

ENG-796 One blue commit colour

BEFORE: the portion sheet's commit CTA is green — colliding with the reserved win/success green. AFTER: the commit CTA is brand blue. Green is reserved for the calorie ring under-target + success only.

ENG-797 One brand mark

BEFORE: a generic-tracker "Log a meal" header. AFTER: a small Suppr mark on the sheet header and portion sheet, so every logging surface reads as the same branded product, not a stock list.

ENG-798 Reserved win-colour / legible confidence

BEFORE: a ~14px green tick is the only confidence signal — nearly invisible and it fights the green macro "P" letters, so "verified" and "protein" share one colour. AFTER: a tinted chip — soft-blue Verified / amber Estimated — that is legible and never collides with the macro palette or the win-colour.

ENG-799 Branded action-sheet

BEFORE: two competing filter languages (Recent│Library│Saved-meals tabs vs All／Recents／Custom／Branded／Generic pills) on adjacent screens. AFTER: one unified segmented control, and the portion sheet is a branded action-sheet with cream surface, grab handle and brand mark.

Unification removes the second filter grammar entirely — one control, one mental model.

Header parity

Title, X, 4-slot Breakfast／Lunch／Dinner／Snacks selector and the search field (barcode · mic · camera) are held constant across BEFORE and AFTER so this reads as the same Log-a-meal sheet — only the results body and confidence signal change.
