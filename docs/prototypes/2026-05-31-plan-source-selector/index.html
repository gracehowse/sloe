What this changes

One shared control, two platforms. A "Plan from" selector (My library / Library & discovery / Discovery only) on both the mobile and web Plan tabs, defaulting to Library & discovery. Pool-building runs through one shared helper (selectPlanPool) so the two platforms can't silently diverge again.

It folds in the ENG-788 empty state. Since 0 saved is no longer a dead end, the held PlanEmptyState becomes the "My library is empty" sub-case, not the whole screen. The selector defaults to a source that works, generate stays live, and "Browse recipe library" survives as a quiet secondary link. ENG-788's primary-accent pill fix carries over unchanged.

It closes the real divergence. Today web pools saved-only and mobile pools saved+discover — invisible drift. The selector makes the pool an explicit, identical choice on both.

Two red-lines for you: (1) Is Library & discovery the right default, or should a brand-new user default to Discovery only until they've saved anything? (2) Happy folding ENG-788's empty state into this (one combined sim review), or keep them as two separate ramps?
