## Shared nutrition helpers (`src/lib/nutrition/`)

Pure (no React, no Supabase, no Date side-effects — callers pass `now`). Imported by both web and mobile so behaviour cannot drift.

| Helper | Purpose |
|--------|---------|
| `mealSlots.ts` | **Audit L5 (2026-04-18)** — Single source of truth for meal-slot names. Exports `MEAL_SLOTS = ["Breakfast", "Lunch", "Dinner", "Snacks"] as const`, the `MealSlot` union, `isMealSlot(s)` (strict guard — exact canonical spelling), and `normaliseMealSlot(raw)` (case-insensitive + trim + legacy `"Snack"` → `"Snacks"`; returns `null` for unrelated strings). Every comparison against a slot name routes through these helpers so a future rename (e.g. localisation) lands in one place. UI rendering literals stay as literals — only *comparison* sites moved. Consumers: `foodHistory.computeEatAgainForSlot`, `savedMeals.normaliseSlot`, web `NutritionTracker` save-combo guard, mobile `(tabs)/index.tsx` save-combo guard, mobile `(tabs)/planner.tsx` `swapMeal` slot-ratio + recipe-fit, mobile `recipe/[id].tsx` `journalSlotFromMealTypes`. |
| `eatAgainDismiss.ts` | **Audit L4 (2026-04-18)** — Shared "Eat again" banner dismiss helper. v2 storage shape `{ dateKey: "YYYY-MM-DD", dismissedAt: ISO-8601 }` under key `STORAGE_KEY = "suppr-eat-again-dismissed-v2"`. `shouldShowEatAgain(stored, now)` hides the banner when `stored.dateKey === today` OR `|now - dismissedAt| < DISMISS_ROLLBACK_WINDOW_MS` (12h) — the rollback window is a belt-and-braces safety net against device clock changes (timezone travel, wobble, manual date change). `recordDismiss(now)` builds a v2 record. `parseDismissState` / `serialiseDismissState` own the JSON round-trip. `migrateLegacyDismiss(v1Raw, now)` lifts the pre-L4 bare-dateKey string (legacy key `"suppr-eat-again-dismissed"`) into a v2 record stamped at `now`. `readDismissState(v2Raw, v1Raw, now)` picks v2 first, falls back to v1. Persistence is host-owned — web uses `localStorage`, mobile uses `AsyncStorage`. |
| `streakReset.ts` | **Ship L6 G8 (2026-04-18)** — Pure predicate `didStreakReset(prior: number \| null, current: number): boolean`. Returns true iff the protected streak transitioned from `>=1 → 0`. Called from a `useEffect` that diffs the prior ref against the latest `computeProtectedStreak().streakLength` on both Today hosts (`NutritionTracker.tsx` web / `app/(tabs)/index.tsx` mobile). Fires `streak_reset { priorStreak }` once per transition. Seeded with `null` so zero-streak first renders never emit a spurious event. |
| `analytics/firstLog.ts` | **Ship L6 G2 (2026-04-18)** — Shared first-log signal helpers. `firstLogTimestamp(now?)` returns the ISO string written as the PostHog `first_log_at` person property. `shouldMarkFirstLog(localMarker)` gates the write on a local dedupe marker (`FIRST_LOG_LOCAL_KEY`). Web `track()` uses `posthog.setPersonProperties({}, { first_log_at })` (the second arg is `$set_once`, idempotent). Mobile `track()` writes via `identify(distinctId, { first_log_at })` gated by an AsyncStorage flag. Both are fire-and-forget; a storage denial never blocks a log. |
