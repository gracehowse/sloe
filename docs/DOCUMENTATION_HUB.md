# Documentation Hub

This is the single source of truth for the product.

## Sections
- `product/` Feature and behaviour documentation
- `user/` End-user guides
- `journeys/` End-to-end user journeys
- `technical/` Architecture and implementation notes
- `api/` API behaviour and contracts
- `data/` Schemas, entities, validation, and integrity rules
- `ui/` UI patterns, states, and design behaviour
- `testing/` Test plans, cases, and regression coverage
- `security/` Auth, permissions, and security controls
- `operations/` Internal processes and support guidance
- `decisions/` Product and technical decision logs

## Rules
- Documentation must reflect reality
- Any meaningful code change must update docs
- No feature is complete without docs
