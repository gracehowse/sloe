# Sloe Kitchen v1 approved content pack

This directory contains only the 18 recipe photographs Grace explicitly approved during visual review and the original Sloe Kitchen recipes written to reproduce them.

## Files

- `recipes.json` — yields, weighed ingredients, cook times, executable steps, allergens and image paths.
- `catalogue.json` — the matching dish and photography briefs for the approved set only.
- `image-provenance.json` — built-in generation provenance, selection notes and the final prompt profile.
- `nutrition.json` — per-serving macros and per-ingredient verification audit generated by Suppr's canonical nutrition engine.
- `hosted-images.json` — the deterministic public Supabase Storage locations used by Discover.
- `images/*.jpg` — 1200 × 1200, app-ready JPEG hero images.

## Content status

- Photographs: founder-approved.
- Recipes: original wording, canonical-technique based and cookability-reviewed against the photographs.
- Nutrition: calculated from the weighed ingredients by `verifyIngredients`; every row clears the acceptance floor and carries its match provenance. No hand-entered macro estimates are published.
- Discovery: `scripts/build-sloe-kitchen-discover-seed.mjs` generates the shared web/mobile static catalogue at `src/lib/recipes/sloeKitchenSeedData.ts`.
- Physical kitchen testing: not represented as completed.

Rejected and unreviewed photographs are not part of this pack. The four rejected workspace JPEGs that predated the final selection were moved to `/private/tmp/sloe-kitchen-rejected-20260719`; their original generated PNGs remain in Codex generation history.

## Validation contract

Every recipe must have a unique slug, a matching image, positive gram weights, a positive yield and cook time, 4–8 steps, and an explicit allergen array. The approved image, recipe and provenance slug sets must be identical.

When this pack changes, run these in order:

1. `npm run sloe-kitchen:verify` — regenerate accepted nutrition from the weighed ingredients.
2. `npm run sloe-kitchen:upload-images` — upsert the approved JPEGs and refresh their public URL manifest.
3. `npm run sloe-kitchen:build-seed` — regenerate the shared web/mobile Discover module.
